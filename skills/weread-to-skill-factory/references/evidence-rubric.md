# 证据评分标准（Evidence Rubric）

每条蒸馏结论必须有来源追踪。没有证据的结论 = 臆测。

## 证据等级

| 等级 | 来源 | 权重 | 说明 |
|------|------|------|------|
| **A** | 个人划线 + 批注 | 最高 | 你读到这句时停下来划线，还写了想法 |
| **B** | 个人划线（无批注） | 高 | 你划了线但没写想法 |
| **C** | 热门划线（>1000人划线） | 中 | 群体共识，但不是你个人认可 |
| **D** | 热门划线（<1000人） | 低 | 少数人关注 |
| **E** | 书籍简介/目录 | 最低 | 元数据，非内容证据 |
| **F** | 推导/综合 | 标注 | 从多个证据推导出的结论 |

**关键原则**：C/D/E 级证据只能作为辅助，不能作为核心模型的唯一来源。
核心模型必须有 A 或 B 级证据支撑。

## 证据密度要求

| 结论类型 | 最低证据 | 理想证据 |
|---------|---------|---------|
| 核心模型 | 1条B级 + 1条C级 | 2条A级 + 2条C级 |
| 决策启发式 | 1条A级 | 2条A级 + 1条D级 |
| 反模式 | 1条A级或B级 | 2条A级 |
| 行动协议 | 1条A级 + 推导 | 2条A级 + 1条C级 |
| 诊断问题 | 推导 | 1条A级 + 推导 |

## 证据台账格式

每个 Level 2/3 的 book-skill 都必须生成 `references/evidence-ledger.md`。
完整模板见 `references/evidence-ledger-template.md`。

## 降级规则

- 如果一条结论只有 E/F 级证据 → 降级为「参考」，不作为核心模型
- 如果一条结论与用户批注矛盾 → 以用户批注为准
- 如果热门划线与个人划线冲突 → 以个人划线为准
- 如果所有核心模型都只有 C 级以下证据 → 该书不适合生成 skill
