# Post-Viva Scoring Protocol

Viva 完成后的结构化评分输出模板。

## 触发条件

Guided Viva 或 Strict Viva 的 10 题全部回答完毕后，执行此 Protocol。

## 输出格式（必须严格遵循）

### 第 1 节：模式确认

```
- 当前书：《书名》
- 当前阶段：[Viva模式] Viva completed → Scoring
- 禁止写文件：✅
- 禁止创建 book-skill：✅
- 禁止安装 skill：✅
```

### 第 2 节：逐项评分（30 分制）

逐维度输出，每个维度包含：
- 维度名 + 分值范围
- 引用用户回答中的关键原文（不超过 2 句）
- 简短评分理由（1-2 句）
- **得分：N** — 一句话总结

评分维度（按 `private-reading-viva.md` 口试评分表）：

| 维度 | 分值 | 评分标准 |
|------|------|---------|
| 核心问题理解 | 0-4 | 0=说不出, 2=大致方向, 4=一句话说清 |
| 概念准确性 | 0-4 | 0=说不出, 2=模糊, 4=准确解释3-5个概念及关系 |
| 结构理解 | 0-4 | 0=说不出, 2=逻辑不清, 4=清晰描述论证路径 |
| 反模式识别 | 0-4 | 0=说不出, 2=宽泛, 4=具体说出作者反对什么 |
| 个人判断 | 0-5 | 0=无, 2=有认可无理由, 3=有理由无保留, 5=有认可+理由+保留+批判 |
| 现实迁移 | 0-5 | 0=无, 2=「有启发」无场景, 3=有场景未实施, 5=真实例子 |
| skill化潜力 | 0-4 | 0=说不出, 2=模糊, 4=清晰触发条件+不适用场景+边界 |

### 第 3 节：一票否决检查

表格形式，逐项检查：

| 检查项 | 结果 |
|--------|------|
| 是否只有书评腔/鸡汤腔 | ❌否 / ✅是 |
| 是否说不出作者反对什么 | ❌否 / ✅是 |
| 是否说不出不同意或保留点 | ❌否 / ✅是 |
| 是否说不出真实使用场景 | ❌否 / ✅是 |
| 是否无法举出自己如何使用这本书思想 | ❌否 / ✅是 |

结论：一票否决：无触发 / 触发 [具体项]

### 第 4 节：阅读痕迹检查

表格形式：

| 痕迹类型 | 具体内容 |
|----------|----------|
| 具体角色/人物 | ... |
| 具体情节/事件 | ... |
| 具体概念/口号/方法 | ... |
| 具体结构变化 | ... |
| 具体个人记忆点 | ... |

结论：阅读痕迹：N个，充分/不足

### 第 5 节：Level 判定

```
总分：X + X + X + X + X + X + X = XX/30
```

对照判定表：

| 分数 | 判定 | 输出级别 |
|------|------|---------|
| 24-30 | ✅ 强通过 | Level 3: Installed Book Skill |
| 19-23 | ⚠️ 条件通过 | Level 2: Draft Book Skill + 要求补充材料 |
| 13-18 | ⚠️ 不足 | Level 1: Reading Card |
| 0-12 且无阅读痕迹 | ❌ 拒绝 | Level 0: Reject |
| 0-12 但有 ≥2 个阅读痕迹 | ⚠️ 阅读痕迹不足 | Refresher Needed |

结论：Level N: [名称] 候选

### 第 6 节：是否允许进入 Draft Book Skill

- 如果不允许：说明缺什么（具体维度 + 需要补充的内容）
- 如果允许：输出 Draft Book Skill 生成计划（目录结构 + 核心模型提取方向）
- **只输出计划，不写文件**

### 第 7 节：批准询问

如果允许进入 Draft Book Skill，最后必须询问：

> "是否批准进入 WRITE_APPROVED，生成《书名》Draft Book Skill？"

## 禁止事项

- 不要重新问 10 题
- 不要输出泛泛读书建议
- 不要跳到写读书笔记
- 不要创建任何文件
- 不要安装 skill
- 不要输出评分理由之外的冗余内容

## 实战经验

### Guided Viva 中断恢复

Guided Viva 跨多轮对话时，可能因上下文压缩或内容过滤中断。用户说「继续」时：
- 不要重新出题
- 不要重复已评分的内容
- 直接从中断处继续（下一题或评分阶段）

### 评分粒度

评分理由应引用用户回答中的关键原文，但不要大段复制。每个维度 1-2 句理由即可。

### 结构理解维度的常见低分

大多数读者能说出核心概念和个人判断，但「结构理解」维度常得 2 分（逻辑不清）。这是因为：
- 传记类书籍的「论证结构」不像学术著作那样显式
- 读者倾向于记住人物和事件，而非叙述逻辑
- 评分时对此维度可适当放宽：能说出「从 X 到 Y 到 Z 的发展脉络」即可给 3 分
