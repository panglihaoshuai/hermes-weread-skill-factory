# Reading Gate 详细评分规则

## WeRead 证据路径（20 分制）

### 1. 阅读完成度（0-4 分）

| 分数 | 条件 |
|------|------|
| 0 | 未读 / 只打开了封面 |
| 1 | 阅读进度 < 25% |
| 2 | 阅读进度 25-50% |
| 3 | 阅读进度 50-75% |
| 4 | 阅读进度 > 75% 或已标记读完 |

数据来源：`/book/getprogress` 或 `/user/notebooks` 的 `markedStatus`/`readingProgress`

### 2. 个人划线密度（0-4 分）

| 分数 | 条件 |
|------|------|
| 0 | 无个人划线 |
| 1 | 1-5 条划线 |
| 2 | 6-15 条划线 |
| 3 | 16-30 条划线 |
| 4 | > 30 条划线 |

数据来源：`/book/bookmarklist` 的 `updated` 数组长度

### 3. 个人批注质量（0-5 分）

| 分数 | 条件 |
|------|------|
| 0 | 无个人批注/想法 |
| 1 | 有批注，但只是简单标记（如「好」「赞」） |
| 2 | 有感悟型批注（表达感受、认同） |
| 3 | 有质疑型批注（提出不同观点、反问） |
| 4 | 有迁移型批注（联想到自己的项目/经历） |
| 5 | 有系统性思考（多条批注形成完整思考链） |

数据来源：`/review/list/mine` 的 `reviews` 内容分析

### 4. 章节覆盖度（0-3 分）

| 分数 | 条件 |
|------|------|
| 0 | 划线/批注只集中在 1 个章节 |
| 1 | 覆盖 2-3 个章节 |
| 2 | 覆盖 4-6 个章节或多处分散 |
| 3 | 覆盖全书主要章节（> 60% 章节有划线或批注） |

数据来源：`/book/bookmarklist` 的 `chapters` 与 `updated` 交叉分析

### 5. 现实连接度（0-4 分）

| 分数 | 条件 |
|------|------|
| 0 | 批注中无任何与现实的连接 |
| 1 | 批注中提及现实场景但无深入 |
| 2 | 批注中有联想（「这让我想到...」） |
| 3 | 批注中有应用（「我可以用这个来...」） |
| 4 | 批注中显示已改变行为/决策（「我决定...」「之前我...现在我...」） |

数据来源：`/review/list/mine` 的内容语义分析

### Gate 判定

| 分数 | 判定 | 输出 |
|------|------|------|
| 16-20 | ✅ 强通过 | Level 3: Installed Book Skill |
| 12-15 | ⚠️ 条件通过 | Level 2: Draft Book Skill + 要求补充 |
| 8-11 | ⚠️ 不足 | Level 1: Reading Card；**如果用户声称精读过 → 转入 Private Reading Viva** |
| 0-7 且有个人批注/用户声称精读/外部证据 | ⚠️ 转入口试 | 转入 Private Reading Viva |
| 用户说「读过但忘了/答不上来/题太多」 | ⚠️ Refresher Needed | Level 1 Reading Card 或 Guided Viva |
| Viva ≤12 但有 ≥2 个阅读痕迹 | ⚠️ 阅读痕迹不足 | Refresher Needed（不得 Level 0 Reject） |
| 0-7 且无任何个人证据且用户不能自证 | ❌ 拒绝 | Level 0: Reject |

**低分处理核心原则**：
- 低分不得自动 Reject
- 低分 + 有个人证据或用户自证 → 必须给 Viva 机会
- 低分 + 无证据 + 不能自证 → 才 Reject
- UNKNOWN 维度不得按 0 分处理（详见 weread-data-contract.md）

### UNKNOWN 归一化规则

当某些评分维度返回 UNKNOWN 时，按以下规则处理：

**1. 从分母中排除 UNKNOWN 维度**

```
known_score = 已知维度得分之和
known_max = 已知维度满分之和
unknown_count = UNKNOWN 维度数量
total_dims = 5（阅读完成度/划线密度/批注质量/章节覆盖/现实连接）
```

**2. 展示格式**

```
已知维度得分：known_score / known_max（unknown_count 个维度 UNKNOWN）
归一化到 20 分制：round(known_score / known_max × 20) / 20
```

**3. UNKNOWN 比例限制**

- UNKNOWN 维度 ≤ 40%（即 ≤ 2/5）→ 正常按归一化分数判定 Level
- UNKNOWN 维度 > 40%（即 ≥ 3/5）→ **不得直接判定 Level 3**，最高只能 Level 2 或进入 Viva
- 全部 5 个维度都是 UNKNOWN → 不评分，直接要求用户提供自证或进入 Viva

**4. 禁止事项**

- UNKNOWN 不得记为 0 分
- UNKNOWN 不得直接 PASS
- UNKNOWN 维度必须在报告中标注原因（「API 未测试」「API 返回错误」「数据不可用」）

## 私下精读自证路径（30 分制）

### 1. 核心问题理解（0-4 分）

| 分数 | 标准 |
|------|------|
| 0 | 说不出核心问题 |
| 2 | 能说出大致方向但不精确 |
| 4 | 一句话说清核心问题，且与书的实际内容吻合 |

### 2. 概念准确性（0-4 分）

| 分数 | 标准 |
|------|------|
| 0 | 说不出任何概念 |
| 2 | 能列出概念但解释模糊 |
| 4 | 能准确解释 3-5 个概念及其关系 |

### 3. 结构理解（0-4 分）

| 分数 | 标准 |
|------|------|
| 0 | 完全说不出论证结构 |
| 2 | 能说「大概讲了什么」但逻辑不清 |
| 4 | 能清晰描述从问题到结论的论证路径 |

### 4. 反模式识别（0-4 分）

| 分数 | 标准 |
|------|------|
| 0 | 说不出作者反对什么 |
| 2 | 能说出作者反对某个宽泛的东西 |
| 4 | 能具体说出作者反对的错误观点/行为/世界观 |

### 5. 个人判断（0-5 分）

| 分数 | 标准 |
|------|------|
| 0 | 无个人判断，只有「书好」 |
| 2 | 有认可但无理由 |
| 3 | 有认可+理由，但无保留意见 |
| 5 | 有认可+理由+保留意见+独立批判 |

### 6. 现实迁移（0-5 分）

| 分数 | 标准 |
|------|------|
| 0 | 无任何现实连接 |
| 2 | 提及「对我有启发」但无具体场景 |
| 3 | 说出具体应用场景但未实施 |
| 5 | 举出真实例子：用这本书的思想做过判断或改变行动 |

### 7. Skill 化潜力（0-4 分）

| 分数 | 标准 |
|------|------|
| 0 | 说不出触发条件和不适用场景 |
| 2 | 能模糊说出「什么时候用」 |
| 4 | 能清晰说出触发条件+不适用场景+边界 |

### 口试判定

| 分数 | 判定 | 输出 |
|------|------|------|
| 24-30 | ✅ 强通过 | Level 3 |
| 19-23 | ⚠️ 条件通过 | Level 2 |
| 13-18 | ⚠️ 不足 | Level 1 |
| 0-12 | ❌ 拒绝 | Level 0 |

### 一票否决项

任一触发 → 直接 Level 0：
- 回答只有书评腔/鸡汤腔/营销号腔
- 只能说「这本书讲得很好，很有启发」
- 说不出作者反对什么
- 说不出自己不同意什么
- 说不出真实使用场景
- 无法举出自己如何用这本书改变判断或行动
