# Private Reading Viva（私下精读口试）

## 触发条件

微信读书缺少用户个人划线/批注，但用户声称自己在其他渠道精读过（纸质书、Kindle、PDF、Obsidian、Notion、课堂、实体笔记等）。

## Viva 模式选择

| 模式 | 触发条件 | 题数 | 评分 | 用途 |
|------|---------|------|------|------|
| **Strict Viva** | 用户说「来吧」「正式口试」 | 10 题（可一次性或分批） | 30 分制 | 正式验收 |
| **Guided Viva** | 用户说「记不太清」「一题一题来」 | 逐题追问，每题最多 3 轮 | 最终按 30 分制 | 引导式回忆 |
| **Fixture Viva** | dry-run 测试 | 模拟答案 | 验证评分器 | 测试模式 |
| **Refresher Needed** | 用户说「忘了/答不上来/题太多/需要回忆」 | 不进入口试 | 不评分 | 输出 Refresher Card 或 Guided Viva |

### 模式选择规则

1. 用户明确要求正式口试 → **Strict Viva**
2. 用户表示记忆模糊、想慢慢来 → **Guided Viva**
3. 用户说「测试」「dry-run」→ **Fixture Viva**
4. 用户说「忘了」「答不上来」「题太多」「需要先回忆一下」→ **Refresher Needed**
5. 不确定时 → 默认 **Guided Viva**（比 Strict 温和，比 Fixture 有效）

---

## 口试问题（固定 10 题，三种模式通用）

1. 用你自己的话说，这本书真正要解决的**核心问题**是什么？
2. 这本书最重要的 **3-5 个概念**是什么？不要只列名词，要解释它们之间的关系。
3. 这本书的**论证结构**是什么？它大概是如何从问题走到结论的？
4. 作者主要**反对**什么错误观点、错误行为或错误世界观？
5. 你最**认可**这本书的哪 3 个判断？为什么？
6. 你**不认可、保留或认为过时**的地方是什么？
7. 这本书对你现在的**项目、投资、学习、训练、教学或人生规划**有什么具体影响？
8. 如果把这本书变成 skill，它应该在什么场景下被**调用**？
9. 它**不应该**在什么场景下被调用？
10. 给出一个你**亲自用这本书的思想做过判断或改变行动**的例子。

---

## Strict Viva（正式验收）

一次性或分批出 10 题。用户回答后按 30 分制评分。

**约束**：
- 可以一次性 10 题，也可以分批出题
- 评分必须基于完整的 10 题答案
- 不完整答案不得 Level 3（最高 Level 2）
- 一票否决项同样适用

---

## Guided Viva（引导式口试）

一次只问 1 题，从 Q1 开始。用户回答后追问，10 题问完后评分。

**允许**：
- 追问（「能再具体一点吗？」）
- 澄清（「你说的 X 是指什么？」）
- 要求举例（「能举个你自己的例子吗？」）
- 要求用户解释自己的判断（「你为什么这么认为？」）

**禁止**：
- 提供标准答案
- 总结剧情
- 解释书中概念
- 暗示正确方向
- 用热门划线提醒用户
- 替用户组织答案

**追问规则**：
- 每题最多追问 3 轮
- 用户说「不知道」→ 记录 0 分，跳到下一题
- 用户回答模糊 → 追问「能举个具体的例子吗？」
- 用户回答可能有误 → 不纠正，追问「你为什么这么理解？」
- 10 题问完后，按同样的 30 分制评分
- 一票否决项同样适用

---

## Fixture Viva（测试模式）

使用模拟用户答案测试评分器。不接受真实用户输入。

**约束**：
- 只用于测试评分器一致性
- 结果不得作为真实用户 Gate 结果
- 不得生成真实 book-skill
- 不得进入 Write-approved Run
- 输出必须标注「Fixture 模拟数据，不代表真实评估」

---

## Refresher Needed（读过但遗忘）

### 触发条件

用户说了以下任何一句：
- 「我读过但忘了」
- 「我现在答不上来」
- 「题太多」
- 「需要先回忆一下」
- 回答明显是现编的、没有细节的

### 处理规则

- **不得判定为伪读**（不是一票否决）
- **不得生成 skill**
- **不得直接 Level 3**
- 输出两种选择让用户选：
  1. **Refresher Card**：基于现有证据（热门划线、简介、章节目录）生成概览，帮助用户回忆
  2. **Guided Viva**：一题一题慢慢来，帮用户重建记忆

### Refresher Card

Refresher Card 用于读过但遗忘的用户恢复记忆。

- **不是 skill**，不安装，不进入 skill 系统
- 输出路径：`~/.hermes/reading-cards/<book-slug>-refresher.md`
- 内容：核心问题、关键概念、章节结构、热门划线 Top 5、建议回忆方向
- 目的：帮用户回忆后重新进入 Guided Viva 或 Strict Viva

---

## 口试评分（30 分制，三种模式通用）

| 维度 | 分值 | 评分标准 |
|------|------|---------|
| 核心问题理解 | 0-4 | 0=说不出, 2=大致方向, 4=一句话说清 |
| 概念准确性 | 0-4 | 0=说不出, 2=模糊, 4=准确解释3-5个概念及关系 |
| 结构理解 | 0-4 | 0=说不出, 2=逻辑不清, 4=清晰描述论证路径 |
| 反模式识别 | 0-4 | 0=说不出, 2=宽泛, 4=具体说出作者反对什么 |
| 个人判断 | 0-5 | 0=无, 2=有认可无理由, 3=有理由无保留, 5=有认可+理由+保留+批判 |
| 现实迁移 | 0-5 | 0=无, 2=「有启发」无场景, 3=有场景未实施, 5=真实例子 |
| skill化潜力 | 0-4 | 0=说不出, 2=模糊, 4=清晰触发条件+不适用场景+边界 |

## 口试判定

| 分数 | 判定 | 输出级别 |
|------|------|---------|
| 24-30 | ✅ 强通过 | Level 3 Candidate（允许进入 Draft + eval；安装仍需 eval PASS + 用户二次批准） |
| 19-23 | ⚠️ 条件通过 | Level 2: Draft Book Skill + 要求补充材料 |
| 13-18 | ⚠️ 不足 | Level 1: Reading Card |
| 0-12 且无阅读痕迹 | ❌ 拒绝 | Level 0: Reject |
| 0-12 但有 ≥2 个阅读痕迹 | ⚠️ 阅读痕迹不足 | Refresher Needed（不得 Level 0 Reject） |

**Level 3 Candidate 说明**：Level 3 Candidate 只代表 Gate/Viva 达到生成 Draft Book Skill 的资格。未经 Draft 生成、eval 通过、用户二次批准，不得称为 Installed。

**不完整答案修正**：如果 Strict Viva 中用户未回答全部 10 题，最高只能 Level 2。

### 阅读痕迹覆盖规则

当 Viva 总分 ≤ 12 时，检查回答中是否包含具体阅读痕迹。

**具体阅读痕迹**包括：
- 具体角色名（如「拿破仑」「雪球」）
- 具体情节片段（如「七诫被篡改」「风车之战」）
- 具体口号/概念（如「所有动物平等，但有些动物更平等」）
- 具体结构变化（如「从平等到猪掌权」）
- 具体个人记忆点（如「读到某个情节时的感受」）

**判定规则**：
- 阅读痕迹 ≥ 2 个 → **Refresher Needed**，不得 Level 0 Reject
- 阅读痕迹 = 0 个 + 回答只有书评腔/鸡汤腔 → **Level 0 Reject**
- 阅读痕迹 = 1 个 → 边界情况，倾向 Refresher Needed

**Refresher Needed 处理**：
- 不得判定为伪读
- 不得 Level 3
- 不得生成 book-skill
- 输出 Refresher Card 或 Guided Viva 选项

## 一票否决项（任一触发 → 直接 Level 0）

**但**：如果回答中有 ≥2 个具体阅读痕迹，一票否决不适用，改为 Refresher Needed。

一票否决只在回答**完全没有具体阅读痕迹**时生效：
- 回答只有书评腔、鸡汤腔、营销号腔
- 只能说「这本书讲得很好，很有启发」
- 说不出作者反对什么
- 说不出自己不同意什么
- 说不出真实使用场景
- 无法举出自己如何用这本书改变判断或行动

## Viva Progress Display（进度显示）

Guided Viva 每次提问必须显示：

```
【Guided Viva｜《书名》｜Qx/10｜剩余N题】
当前阶段：Viva in progress
当前状态：GUIDED_VIVA_RUNNING
```

**格式要求**：
- 书名必须显示
- 当前题号：Qx/10
- 剩余题数：10-x
- 当前阶段：Viva in progress
- 当前状态：GUIDED_VIVA_RUNNING

**禁止**：
- 不显示进度直接提问
- 只显示题号不显示剩余
- 跳过进度显示

---

## Post-Viva Scoring Protocol（评分协议）

当 Q10 回答完成后，必须立即切换状态：

```
GUIDED_VIVA_COMPLETED → POST_VIVA_SCORING
```

**必须输出**：

1. 7 个维度逐项评分（0-4 或 0-5）
2. 总分 /30
3. 一票否决检查（5 项）
4. 阅读痕迹检查（5 类）
5. Level 判定（Level 0/1/2/3 Candidate）
6. 是否允许进入 Draft Book Skill
7. 如果允许，询问用户是否批准 WRITE_APPROVED 生成 Draft Book Skill

**禁止**：
- 不得只输出「你阅读很扎实」
- 不得跳到「建议写读书笔记」
- 不得忘记 Level 判定
- 不得忘记询问是否生成 Draft Book Skill

**评分完成后状态切换**：

```
POST_VIVA_SCORING → WAITING_WRITE_APPROVAL
```

---

## Viva Completion Guard（完成守卫）

如果已经完成 Q10，但尚未评分（状态为 GUIDED_VIVA_COMPLETED）：

系统必须拒绝：
- 新话题
- 自由聊天
- 读书建议
- 其他 skill

并提示：

「Viva 已完成但尚未评分。必须先执行 Post-Viva Scoring。」

**禁止**：离开当前状态。

---

## High-risk Truncation Handling（截断处理）

如果某个问题或回答触发：
- high risk
- rejected
- truncated

**禁止**：
- 中断整个 Viva
- 退出 skill
- 丢失状态机

**必须**：

将该题标记为：
- `SAFE_REPHRASE`（用安全表述重问同一考点）
- 或 `SKIPPED`（无法重问则跳过）
- 或 `UNKNOWN`（记录为未知）

然后继续后续题目。

**不得因此把整场 Viva 判 FAIL。**

状态机必须保持：`GUIDED_VIVA_RUNNING`
